# OpenClaw-Workspace 部署指南

## 项目信息

- **框架**: Nuxt 3 + Nuxt UI + UnoCSS + Pinia
- **路径**: `/root/.openclaw/workspace/OpenClaw-Workspace`
- **分支**: `feature/ui-refinement`

## 生产部署

### ⚠️ 重要：禁止使用开发模式
**不要使用 `npm run dev` 启动服务！** 开发模式性能差且不稳定。

### 启动生产服务器
```bash
cd /root/.openclaw/workspace/OpenClaw-Workspace

# 1. 停止所有 Nuxt 进程
pkill -f "nuxt" || true

# 2. 构建项目（首次或代码更新后执行）
npm run build

# 3. 启动生产服务器（后台运行）
nohup node .output/server/index.mjs > /tmp/nuxt-prod.log 2>&1 &

# 4. 查看日志
tail -f /tmp/nuxt-prod.log
```

### 快速重启（已构建）
```bash
cd /root/.openclaw/workspace/OpenClaw-Workspace
pkill -f "nuxt" || true
sleep 2
nohup node .output/server/index.mjs > /tmp/nuxt-prod.log 2>&1 &
```

### 开发调试（仅用于本地开发）
⚠️ **生产环境禁止使用以下命令**
```bash
# 仅在本地开发时使用
npm run dev -- --host 0.0.0.0 --port 3000
```

## 部署架构

### 服务架构
- **反向代理**: Traefik (Docker)
- **域名**: `dev.theirises.cn` → `10.10.0.11:3000`
- **SSL**: Let's Encrypt 自动 HTTPS
- **认证**: 当前无认证

### Traefik 路由配置 (dynamic.yml)
```yaml
# OpenClaw Workspace - 子域名 dev.theirises.cn
openclaw-workspace-dev:
  rule: "Host(`dev.theirises.cn`)"
  entryPoints:
    - "websecure"
  service: "openclaw-workspace"
  tls:
    certResolver: "letsencrypt"
```

### 服务配置
```yaml
openclaw-workspace:
  loadBalancer:
    servers:
      - url: "http://10.10.0.11:3000"
    passHostHeader: true
```

### 修改 Traefik 配置
配置文件位置: Docker 容器内 `/etc/traefik/dynamic.yml`

```bash
# 查看当前配置
docker exec traefik cat /etc/traefik/dynamic.yml

# 重启 Traefik 生效
docker restart traefik
```

## Nuxt 配置说明

- **base**: `/` (部署在根路径)
- **ssr**: false (SPA 模式)
- **监听地址**: `0.0.0.0:3000` (需让 Docker 可以访问)
